obspy/psn provides read only for the PSN format Volume and  event file structures.  

File structure can be found here:  http://psn.quake.net/psnformat4.html
The registered IRIS network code for the PSN is AM.
Standard PSN settings Azimuth: N = 0, E = 90, Z = 0, Incidence: N=90, E=90, Z=0, Unknown=-12345.
Delay settings: If you are using my Amp / Filter board with a 10 Hz low-pass filter you should use a value 
of around 50 milliseconds. For PSN-ACCEL and PSN-ADC24 boards running at 100 SPS or higher a 
value of 20 milliseconds can be used.
Information from WinSDR documentation.

Specific notes:  The WinQuake calibration number is the reciprocal of the SAC PZ constant and requires
a factor of 100 to get from cm/sec to meters/sec.  When transfering the WinQuake sensitivity to  
trace.stats.calib the factor of 100 is automatically taken into account and the trace.plot() produces the correct plot. 

If pole zero information is not added by your logging system it can be added to the trace using utilities found in the 
SAC IO module as follows:
from obspy import sac
respf = 'C:\\PZ.txt'   #SAC format polezero file
sac.attach_paz(correct[0], respf, tovel=False)  # Normal SAC pz changes to displacement
or 
with a RESP format file:
from obspy import sac
RESPB = 'C:\\RESP.AM.RPSD..MHZ'
with open(RESPB, "r") as fpin:
     attach_resp(correct[0], RESPB, todisp=False, tovel=False, torad=False,
                tohz=False)   # Set switches as desired RESP file is velocity by default.
